# Сценарии проверки AI RealChat

## 1. Переключение режимов (как в ChatGPT)

1. Открыть UI.
2. Убедиться, что доступны режимы `Чат` и `Realtime голос`.
3. Переключиться между ними несколько раз.

Ожидаемо:
- переключение мгновенное;
- история сообщений общая для обоих режимов.

## 2. Smoke для текстового чата

1. В режиме `Чат` отправить текстовый вопрос.
2. Проверить появление ответа ассистента.

Ожидаемо: `POST /api/chat` возвращает `response_text`, сообщение добавляется в общий чат.

## 3. Smoke для голосового режима

1. Открыть UI и нажать `Start speaking`.
2. Сказать фразу на одном из поддерживаемых языков.
3. Нажать `Stop`.
4. Проверить, что появились:
- текст распознавания пользователя;
- текст ответа ассистента;
- голосовой ответ (аудио проигралось).

Ожидаемо: backend вернул `transcript + response_text + audio_base64`.

## 4. Перебивание озвучки (barge-in)

1. Запустить обычный запрос и дождаться, когда ассистент начнет говорить.
2. Пока ответ озвучивается, снова нажать `Start speaking`.
3. Сказать короткое уточнение или поправку.
4. Нажать `Stop`.

Ожидаемо:
- озвучка прошлого ответа сразу останавливается;
- новый запрос отправляется с мета-информацией о перебивании;
- ассистент не повторяет старый ответ полностью, а продолжает по новой реплике.

## 5. Недосказанная фраза при перебивании

1. Во время озвучки ассистента нажать `Start speaking`.
2. Сказать незавершенную фразу, например: `Подожди, а если...`.
3. Нажать `Stop`.

Ожидаемо:
- модель воспринимает реплику как потенциально незавершенную;
- вместо выдумывания деталей задает короткий уточняющий вопрос.

## 6. Смена языка пользователем

1. Сказать запрос на русском.
2. Следующий запрос сказать на английском.
3. Третий запрос сказать на казахском.

Ожидаемо: язык ответа каждый раз совпадает с языком текущего запроса.

## 7. UI язык отдельно от языка диалога

1. Переключить UI на `EN`.
2. Говорить по-русски.
3. Переключить UI на `KZ`.
4. Говорить по-английски.

Ожидаемо:
- язык кнопок/подписей меняется по UI-переключателю;
- язык ответа ассистента определяется по речи пользователя, а не по UI.

## 8. Ошибки и устойчивость

1. Запретить микрофон в браузере.
2. Нажать `Start speaking`.
3. Восстановить доступ и повторить.

Ожидаемо: корректный статус ошибки доступа к микрофону и нормальное восстановление после разрешения.

## 9. Авто-тесты backend

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements-dev.txt
pytest -q
```

Покрытие авто-тестами:
- парсинг и санитизация `history`;
- парсинг и санитизация `turn_meta`;
- эвристика незавершенной реплики;
- прохождение `barge_in` мета-данных через `/api/voice`.
- `POST /api/chat` (текстовый режим) и опциональный `speak=true`.
